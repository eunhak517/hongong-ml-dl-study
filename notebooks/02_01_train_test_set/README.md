## 📌 실습 소개

이 실습에서는 머신러닝 모델을 올바르게 평가하기 위해 전체 데이터를  
**훈련 세트(train set)**와 **테스트 세트(test set)**로 나누는 방법을 학습합니다.

처음에는 도미 데이터만 훈련 세트에 넣고 빙어 데이터만 테스트 세트에 넣습니다.  
그 결과 모델은 도미만 학습했기 때문에 빙어를 전혀 맞히지 못해 정확도 `0.0`이 나옵니다.

이 문제를 해결하기 위해 NumPy로 샘플의 인덱스를 무작위로 섞은 뒤,  
입력 데이터와 정답 데이터를 **같은 인덱스로 함께 선택**합니다.

이 과정을 통해 다음 사실을 확인합니다.

> 데이터를 단순히 앞뒤로 자르면 클래스가 한쪽에 몰릴 수 있으며,  
> 훈련 데이터와 테스트 데이터가 전체 데이터를 고르게 대표해야 한다.

# 🧠 먼저 알아야 할 핵심 개념

## 훈련 세트와 테스트 세트

| 구분 | 역할 |
|---|---|
| 훈련 세트 | 모델이 데이터의 패턴을 학습하는 데 사용 |
| 테스트 세트 | 학습에 사용하지 않은 데이터로 모델을 평가 |
| 입력 데이터 | 생선의 길이와 무게 |
| 타깃 데이터 | 도미 `1`, 빙어 `0` |

머신러닝 모델을 평가할 때는 학습에 사용한 데이터가 아니라  
**처음 보는 데이터**를 사용해야 실제 성능을 더 공정하게 확인할 수 있습니다.

## 샘플링 편향

샘플링 편향은 훈련 세트나 테스트 세트가 전체 데이터의 특성을  
고르게 대표하지 못하고 특정 클래스나 특성에 치우치는 현상입니다.

이 실습의 처음 분할 방식은 다음과 같습니다.

```text
전체 데이터 순서
도미 35마리 → 빙어 14마리

앞 35개를 훈련 세트로 선택
→ 도미만 포함

뒤 14개를 테스트 세트로 선택
→ 빙어만 포함
```

모델은 도미만 학습했기 때문에 빙어를 구분하는 방법을 배울 수 없습니다.

# 🔍 코드 한 줄씩 이해하기

## 1. 생선 길이와 무게 데이터 준비

```python
fish_length = [25.4, 26.3, 26.5, ..., 14.3, 15.0]
```

- `fish_length` 변수에 생선 49마리의 길이를 저장합니다.
- 앞의 35개는 도미의 길이입니다.
- 뒤의 14개는 빙어의 길이입니다.
- 리스트의 값 하나가 생선 한 마리의 길이입니다.

```python
fish_weight = [242.0, 290.0, 340.0, ..., 19.7, 19.9]
```

- `fish_weight` 변수에 같은 생선 49마리의 무게를 저장합니다.
- 길이 리스트와 무게 리스트의 같은 위치는 같은 생선을 나타냅니다.

```text
fish_length[0] = 25.4
fish_weight[0] = 242.0

→ 첫 번째 생선의 길이는 25.4, 무게는 242.0
```

두 리스트의 길이와 순서는 반드시 같아야 합니다.

---

## 2. 입력 데이터와 타깃 만들기

```python
fish_data = [[l, w] for l, w in zip(fish_length, fish_weight)]
```

### `zip(fish_length, fish_weight)`

두 리스트의 같은 위치에 있는 값을 한 쌍씩 묶습니다.

```text
(25.4, 242.0)
(26.3, 290.0)
(26.5, 340.0)
...
```

코드의 `zip (`처럼 함수 이름과 괄호 사이에 공백이 있어도 파이썬 문법상 문제는 없습니다.  
일반적으로는 `zip(`처럼 붙여 쓰는 형태가 더 많이 사용됩니다.

### `for l, w in ...`

- 각 쌍의 첫 번째 값인 길이를 `l`에 저장합니다.
- 두 번째 값인 무게를 `w`에 저장합니다.
- 모든 생선 쌍을 하나씩 반복합니다.

### `[l, w]`

생선 한 마리를 다음 구조로 만듭니다.

```python
[길이, 무게]
```

### 바깥쪽 리스트

49마리의 데이터를 모두 모아 2차원 형태로 만듭니다.

```python
[
    [25.4, 242.0],
    [26.3, 290.0],
    ...
    [15.0, 19.9]
]
```

```python
fish_target = [1] * 35 + [0] * 14
```

- `[1] * 35`는 값 `1`을 35개 만듭니다.
- 도미가 35마리이므로 도미의 정답을 `1`로 표시합니다.
- `[0] * 14`는 값 `0`을 14개 만듭니다.
- 빙어가 14마리이므로 빙어의 정답을 `0`으로 표시합니다.
- `+` 연산자로 두 리스트를 이어 붙입니다.

```text
1 → 도미
0 → 빙어
```

입력과 정답의 순서는 다음처럼 대응합니다.

```text
fish_data[0]   ↔ fish_target[0]
fish_data[1]   ↔ fish_target[1]
...
fish_data[48]  ↔ fish_target[48]
```

## 5. 리스트 슬라이싱으로 일부 데이터 확인

```python
print(fish_data[0:4])
```

- 인덱스 `0`부터 `4` 직전까지 선택합니다.
- 종료 인덱스인 `4`는 포함되지 않습니다.
- 실제로는 `0`, `1`, `2`, `3`번 샘플이 출력됩니다.

```text
시작 인덱스 : 포함
종료 인덱스 : 미포함
```

출력:

```python
[
    [25.4, 242.0],
    [26.3, 290.0],
    [26.5, 340.0],
    [29.0, 363.0]
]
```

```python
print(fish_data[:5])
```

- 시작 인덱스를 생략하면 처음부터 시작합니다.
- 인덱스 `0`부터 `5` 직전까지 선택합니다.
- 처음 5개 샘플이 출력됩니다.

```python
print(fish_data[44:])
```

- 종료 인덱스를 생략하면 마지막까지 선택합니다.
- 인덱스 `44`부터 `48`까지 총 5개를 선택합니다.
- 출력된 샘플은 모두 빙어 데이터입니다.

### 슬라이싱 규칙 정리

| 코드 | 의미 |
|---|---|
| `data[0:4]` | 0번부터 3번까지 |
| `data[:5]` | 처음부터 4번까지 |
| `data[44:]` | 44번부터 끝까지 |
| `data[:]` | 처음부터 끝까지 전체 |

## 6. 잘못된 방식으로 훈련·테스트 세트 나누기

```python
train_input = fish_data[:35]
```

- 처음부터 인덱스 `34`까지 35개의 입력을 선택합니다.
- 전체 데이터에서 앞 35개는 모두 도미입니다.
- 따라서 훈련 입력에는 도미만 들어갑니다.

```python
train_target = fish_target[:35]
```

- 타깃의 앞 35개를 선택합니다.
- 값은 모두 도미를 나타내는 `1`입니다.

```python
test_input = fish_data[35:]
```

- 인덱스 `35`부터 마지막까지 14개의 입력을 선택합니다.
- 이 구간에는 빙어만 들어 있습니다.

```python
test_target = fish_target[35:]
```

- 타깃의 인덱스 `35`부터 마지막까지 선택합니다.
- 값은 모두 빙어를 나타내는 `0`입니다.

최종 구조:

```text
훈련 세트: 도미 35마리
테스트 세트: 빙어 14마리
```

이 분할은 전체 데이터를 고르게 대표하지 못합니다.

## 7. 잘못 분할한 데이터로 모델 훈련 및 평가

```python
kn.fit(train_input, train_target)
```

- 도미 35마리만 사용해 모델을 학습합니다.
- 모델은 훈련 과정에서 클래스 `1`만 확인합니다.
- 빙어 클래스 `0`의 데이터는 전혀 학습하지 못합니다.

```python
kn.score(test_input, test_target)
```

- 테스트 입력인 빙어 14마리를 예측합니다.
- 예측 결과와 실제 타깃인 `0`을 비교합니다.
- 모델은 도미만 학습했기 때문에 빙어도 도미로 예측합니다.

실행 결과:

```text
0.0
```

### 결과 해석

- 14개의 빙어를 하나도 맞히지 못했습니다.
- 정확도 `0.0`은 0%를 의미합니다.
- 알고리즘 자체의 문제가 아니라 **잘못된 데이터 분할**이 원인입니다.

### 핵심 원인

```text
도미만 학습
→ 모델이 빙어의 특징을 본 적이 없음
→ 빙어 테스트 데이터 전부 오분류
```

이 현상이 바로 샘플링 편향의 대표적인 예입니다.

## 12. 무작위 인덱스 만들기

```python
np.random.seed(42)
```

- NumPy의 난수 생성 시작 상태를 `42`로 고정합니다.
- 같은 시드값을 사용하면 코드를 다시 실행해도 같은 무작위 결과가 나옵니다.
- 실험 결과를 재현하기 위해 사용합니다.
- 숫자 `42` 자체에 특별한 머신러닝 의미는 없습니다.

```python
index = np.arange(49)
```

### `np.arange(49)`

- `0`부터 `48`까지 연속된 정수를 생성합니다.
- 마지막 값 `49`는 포함되지 않습니다.
- 전체 49개 샘플의 위치를 나타내는 인덱스 배열입니다.

생성 직후:

```python
[0, 1, 2, 3, ..., 47, 48]
```

```python
np.random.shuffle(index)
```

- `index` 배열의 원소 순서를 무작위로 섞습니다.
- 반환값으로 새 배열을 만드는 것이 아니라 기존 배열 자체를 변경합니다.
- 입력 데이터 자체를 먼저 뒤섞는 대신 인덱스만 섞습니다.

### 인덱스를 섞는 이유

입력 데이터와 타깃은 반드시 함께 움직여야 합니다.

```text
입력 13번 샘플 ↔ 타깃 13번 정답
```

인덱스를 하나만 만들어 입력과 타깃에 똑같이 사용하면 대응 관계가 유지됩니다.

---

## 13. 섞인 인덱스 확인

```python
print(index)
```

실행 결과:

```text
[13 45 47 44 17 27 26 25 31 19 12  4 34  8  3  6 40 41 46 15  9 16 24 33
 30  0 43 32  5 29 11 36  1 21  2 37 35 23 39 10 22 18 48 20  7 42 14 28
 38]
```

- 숫자는 생선 데이터의 원래 위치를 의미합니다.
- 첫 번째로 선택할 샘플은 원래 인덱스 `13`입니다.
- 두 번째는 원래 인덱스 `45`입니다.
- 모든 숫자는 0부터 48까지 한 번씩만 등장합니다.
- 순서만 바뀌었으며 샘플이 추가되거나 사라지지 않았습니다.

---

## 14. 무작위 훈련 세트 만들기

```python
train_input = input_arr[index[:35]]
```

이 코드는 두 단계로 이해할 수 있습니다.

### `index[:35]`

섞인 인덱스 중 처음 35개를 선택합니다.

```text
[13, 45, 47, 44, ..., 5, 29, 11]
```

### `input_arr[...]`

선택한 인덱스 위치의 입력 샘플 35개를 한 번에 가져옵니다.

이처럼 정수 배열을 이용해 여러 위치를 한꺼번에 선택하는 것을  
**배열 인덱싱** 또는 **팬시 인덱싱(fancy indexing)**이라고 부릅니다.

```python
train_target = target_arr[index[:35]]
```

- 입력에 사용한 것과 정확히 같은 35개 인덱스를 타깃에도 사용합니다.
- 따라서 각 입력 샘플과 정답의 대응이 유지됩니다.

```text
train_input[0]  = 원래 input_arr[13]
train_target[0] = 원래 target_arr[13]
```

> 입력과 타깃을 서로 다른 인덱스로 선택하면 정답이 뒤섞여 모델이 잘못 학습합니다.

---

## 15. 배열 인덱싱 결과 확인

```python
print(input_arr[13], train_input[0])
```

### `input_arr[13]`

- 원본 입력 배열의 13번 샘플을 가져옵니다.

### `train_input[0]`

- 새 훈련 입력의 첫 번째 샘플을 가져옵니다.
- 섞인 인덱스의 첫 값이 `13`이므로 원본 13번 샘플과 같아야 합니다.

실행 결과:

```text
[ 32. 340.] [ 32. 340.]
```

두 값이 같은 것은 배열 인덱싱이 의도대로 작동했다는 뜻입니다.

---

## 16. 무작위 테스트 세트 만들기

```python
test_input = input_arr[index[35:]]
```

- 섞인 인덱스에서 35번째 위치부터 끝까지 선택합니다.
- 앞의 35개를 훈련 세트로 사용했으므로 남은 14개를 테스트 세트로 사용합니다.
- 입력 샘플 14개가 만들어집니다.

```python
test_target = target_arr[index[35:]]
```

- 테스트 입력과 동일한 인덱스로 정답을 선택합니다.
- 테스트 입력과 타깃의 대응 관계가 유지됩니다.

최종 크기:

```text
train_input  → 35개 샘플
train_target → 35개 정답
test_input   → 14개 샘플
test_target  → 14개 정답
```

## 17. 훈련 세트와 테스트 세트 시각화

```python
import matplotlib.pyplot as plt
```

- 그래프를 그리기 위해 Matplotlib의 `pyplot`을 가져옵니다.
- `plt`라는 별명을 사용합니다.

```python
plt.scatter(train_input[:, 0], train_input[:, 1])
```

이 줄에서 `:`와 열 인덱스를 이해하는 것이 중요합니다.

### `train_input[:, 0]`

- `:`는 모든 행을 선택한다는 뜻입니다.
- `0`은 첫 번째 열을 선택한다는 뜻입니다.
- 따라서 훈련 세트의 모든 생선 길이를 가져옵니다.

### `train_input[:, 1]`

- 모든 행에서 두 번째 열을 선택합니다.
- 훈련 세트의 모든 생선 무게를 가져옵니다.

```text
train_input[:, 0] → 모든 샘플의 길이
train_input[:, 1] → 모든 샘플의 무게
```

이 두 배열을 x축과 y축으로 사용해 훈련 세트를 산점도로 그립니다.

```python
plt.scatter(test_input[:, 0], test_input[:, 1])
```

- 테스트 세트의 길이와 무게를 같은 그래프에 추가합니다.
- 두 번째 산점도이므로 훈련 세트와 다른 기본 색상으로 표시됩니다.

```python
plt.xlabel('length')
```

- x축 이름을 `length`로 설정합니다.

```python
plt.ylabel('weight')
```

- y축 이름을 `weight`로 설정합니다.

```python
plt.show()
```

- 완성된 그래프를 출력합니다.

### 그래프 해석

훈련 세트와 테스트 세트 양쪽에 도미와 빙어가 섞여 있습니다.  
첫 번째 분할처럼 한쪽에 도미만, 다른 쪽에 빙어만 존재하지 않습니다.

따라서 두 세트가 전체 생선 데이터를 이전보다 더 고르게 대표합니다.

# 💡 이 실습의 핵심 정리

## 1. 입력과 타깃은 반드시 함께 섞어야 합니다

잘못된 방식:

```python
np.random.shuffle(input_arr)
np.random.shuffle(target_arr)
```

입력과 타깃을 각각 따로 섞으면 순서가 서로 달라질 수 있습니다.

안전한 방식:

```python
index = np.arange(49)
np.random.shuffle(index)

input_arr[index]
target_arr[index]
```

같은 인덱스를 사용하면 입력과 정답의 대응이 유지됩니다.

## 2. 데이터 순서가 모델 성능을 왜곡할 수 있습니다

클래스별로 정렬된 데이터를 그대로 앞뒤로 나누면  
훈련 세트와 테스트 세트의 데이터 분포가 완전히 달라질 수 있습니다.

## 3. 테스트 세트는 모델 학습에 사용하면 안 됩니다

테스트 세트는 최종 성능을 확인하는 용도입니다.  
테스트 결과를 보며 모델을 반복적으로 수정하면 테스트 데이터에 맞춰질 수 있습니다.

## 4. 무작위 결과는 재현 가능해야 합니다

np.random.seed(42)

같은 분할을 얻을 수 있도록 난수 시드를 고정합니다.

## 5. 배열의 행과 열을 구분해야 합니다

# ⚠️ 현재 방식의 한계

이 실습에서는 무작위 인덱스를 사용해 데이터를 나누었지만,  
작은 데이터셋에서는 우연히 특정 클래스의 비율이 한쪽에 치우칠 수 있습니다.

현재 전체 클래스 비율:

```text
도미 35마리
빙어 14마리
```

보다 안정적인 분할을 위해 이후에는 scikit-learn의  
`train_test_split()`과 `stratify` 옵션을 사용합니다.

```python
from sklearn.model_selection import train_test_split

train_input, test_input, train_target, test_target = train_test_split(
    input_arr,
    target_arr,
    stratify=target_arr,
    random_state=42
)
```

`stratify=target_arr`는 훈련 세트와 테스트 세트의 클래스 비율을  
전체 데이터와 비슷하게 유지하도록 도와줍니다.
