## 📌 실습 소개

이 실습에서는 생선의 **길이(length)**와 **무게(weight)**를 입력으로 사용해  
생선이 **도미인지 빙어인지 분류하는 머신러닝 모델**을 만듭니다.

사용한 알고리즘은 **k-최근접 이웃 분류기(K-Nearest Neighbors Classifier, KNN)**입니다.

KNN은 새로운 데이터가 들어왔을 때 주변에 있는 가장 가까운 데이터들을 확인하고,  
그중 어떤 클래스가 더 많은지를 기준으로 새로운 데이터의 클래스를 결정합니다.

예를 들어 길이 30cm, 무게 600g인 생선 주변에 도미 데이터가 많이 있다면  
모델은 해당 생선을 도미로 예측합니다.

## 🎯 학습 목표

이 실습을 통해 다음 내용을 이해하는 것을 목표로 합니다.

- 머신러닝에서 **샘플**, **특성**, **타깃**이 무엇인지 이해합니다.
- 도미와 빙어 데이터를 산점도로 시각화합니다.
- 길이와 무게를 하나의 입력 데이터로 결합합니다.
- 정답 데이터인 타깃을 만듭니다.
- `KNeighborsClassifier` 모델을 생성하고 학습합니다.
- 모델의 정확도를 확인합니다.
- 새로운 생선 데이터를 입력해 클래스를 예측합니다.
- `n_neighbors` 값이 분류 결과에 미치는 영향을 확인합니다.
- 훈련 데이터로 평가한 정확도와 실제 일반화 성능은 다를 수 있음을 이해합니다.

- 변수에는 도미 35마리의 길이를 리스트로 저장합니다.
- 리스트 안의 숫자 하나가 도미 한 마리의 길이를 의미합니다.

- 총 35개의 값이 있으므로 도미 샘플도 35개입니다.

```python
bream_weight = [242.0, 290.0, 340.0, ..., 975.0, 950.0]
```

- `bream_weight`라는 변수를 만듭니다.
- 도미 35마리의 무게를 리스트로 저장합니다.

- `bream_length[0]`과 `bream_weight[0]`은 같은 도미의 길이와 무게입니다.


## 6. 길이와 무게를 2차원 입력 데이터로 만들기

```python
fish_data = [[l, w] for l, w in zip(length, weight)]
```

이 한 줄에는 여러 문법이 함께 사용되었습니다.

### `zip(length, weight)`

- `length`와 `weight`에서 같은 위치의 값을 하나씩 묶습니다.
- 다음과 같은 쌍을 차례대로 만듭니다.

```text
(25.4, 242.0)
(26.3, 290.0)
...
(15.0, 19.9)
```

### `for l, w in ...`

- `zip()`이 만든 각 쌍을 반복해서 꺼냅니다.
- 길이는 변수 `l`, 무게는 변수 `w`에 저장합니다.
- `l`은 length의 첫 글자, `w`는 weight의 첫 글자를 사용한 짧은 변수명입니다.

### `[l, w]`

- 길이와 무게를 하나의 리스트로 만듭니다.
- 생선 한 마리의 특성 두 개가 `[길이, 무게]` 형태로 저장됩니다.

### 바깥쪽 `[...]`

- 각 생선의 `[길이, 무게]`를 모두 모아 전체 데이터셋을 만듭니다.
- 이 문법을 **리스트 컴프리헨션(list comprehension)**이라고 합니다.

### 최종 구조

```python
[
    [25.4, 242.0],
    [26.3, 290.0],
    ...
    [15.0, 19.9]
]
```

- 바깥 리스트의 길이는 49입니다.
- 안쪽 리스트 하나가 생선 한 마리입니다.
- 각 생선은 길이와 무게라는 2개의 특성을 가집니다.
- 따라서 데이터의 형태는 개념적으로 `(49, 2)`입니다.

```python
print(fish_data)
```

### 설명

- 만들어진 입력 데이터를 화면에 출력합니다.
- 길이와 무게가 올바르게 묶였는지 직접 확인하기 위한 코드입니다.

## 7. 정답 데이터 만들기

```python
fish_target = [1] * 35 + [0] * 14
```

### `[1] * 35`

- 값 `1`이 들어 있는 리스트를 35번 반복합니다.
- 도미 샘플이 35개이므로 도미의 정답을 모두 `1`로 표시합니다.

```text
[1, 1, 1, ..., 1]  # 총 35개
```

### `[0] * 14`

- 값 `0`이 들어 있는 리스트를 14번 반복합니다.
- 빙어 샘플이 14개이므로 빙어의 정답을 모두 `0`으로 표시합니다.

```text
[0, 0, 0, ..., 0]  # 총 14개
```

### `+`

- 두 리스트를 이어 붙입니다.
- 앞의 35개는 도미, 뒤의 14개는 빙어의 정답이 됩니다.

### 클래스 의미

```text
1 → 도미
0 → 빙어
```

```python
print(fish_target)
```

### 설명

- 만든 정답 리스트를 출력합니다.
- 입력 데이터의 순서와 정답의 순서가 일치하는지 확인합니다.
- `fish_data`와 `fish_target`의 길이는 모두 49여야 합니다.

> 머신러닝에서는 입력 샘플과 정답의 순서가 반드시 일치해야 합니다.

```python
kn = KNeighborsClassifier()
```

### 설명

- `KNeighborsClassifier` 클래스를 이용해 실제 모델 객체를 만듭니다.
- 만든 객체를 `kn`이라는 변수에 저장합니다.
- 별도의 옵션을 입력하지 않았으므로 기본값이 사용됩니다.

이 시점에는 모델 객체만 생성되었을 뿐, 아직 데이터가 전달되지 않았습니다.

## 10. 모델 훈련

```python
kn.fit(fish_data, fish_target)
```

### 설명

- `fit()`은 모델에 입력 데이터와 정답을 전달하는 메서드입니다.
- 첫 번째 인자 `fish_data`는 길이와 무게로 구성된 입력 데이터입니다.
- 두 번째 인자 `fish_target`은 도미와 빙어를 나타내는 정답입니다.
- KNN은 선형 회귀처럼 복잡한 공식을 계산해 계수를 학습하는 방식이 아닙니다.
- 훈련 데이터를 저장해 두었다가 새로운 데이터가 들어오면 가까운 샘플을 찾습니다.

### 출력

```text
KNeighborsClassifier()
```

- 모델이 정상적으로 학습되었음을 보여주는 객체 표현입니다.
- 점수나 예측 결과를 의미하는 것은 아닙니다.

## 11. 정확도 평가

```python
kn.score(fish_data, fish_target)
```

### 설명

- `score()`는 입력 데이터를 예측한 뒤 실제 정답과 비교합니다.
- 분류 모델의 `score()`는 기본적으로 **정확도**를 반환합니다.
- 정확도는 다음과 같이 계산됩니다.

```text
정확도 = 올바르게 예측한 샘플 수 / 전체 샘플 수
```

### 실행 결과

```text
1.0
```

### 결과 해석

- 전체 49개 샘플을 모두 올바르게 분류했다는 뜻입니다.
- 정확도 `1.0`은 100%와 같습니다.

### 반드시 주의할 점

현재는 **학습에 사용한 데이터로 다시 평가**했습니다.

따라서 이 점수만으로 새로운 데이터에서도 100% 정확하다고 말할 수 없습니다.  
모델의 일반화 성능을 확인하려면 이후 학습할 **훈련 세트와 테스트 세트 분리**가 필요합니다.

## 12. 새로운 생선 예측

```python
kn.predict([[30, 600]])
```

### `predict()`

- 학습된 모델로 새로운 입력의 클래스를 예측합니다.

### `[[30, 600]]`

- 길이가 `30`, 무게가 `600`인 생선 한 마리를 의미합니다.
- 안쪽 리스트 `[30, 600]`은 샘플 한 개입니다.
- 바깥 리스트는 여러 샘플을 담는 데이터셋 구조입니다.
- 샘플이 하나여도 scikit-learn은 2차원 형태를 요구하므로 대괄호를 두 번 사용합니다.

```text
잘못된 형태: [30, 600]
올바른 형태: [[30, 600]]
```

### 실행 결과

```python
array([1])
```

### 결과 해석

- 예측 결과 `1`은 도미를 뜻합니다.
- 따라서 길이 30, 무게 600인 생선을 도미로 분류했습니다.
- 결과가 배열인 이유는 여러 샘플의 예측 결과를 한꺼번에 반환할 수 있기 때문입니다.

## 13. 모델이 저장한 입력 데이터 확인

```python
print(kn._fit_X)
```

### 설명

- 학습된 KNN 모델 내부에 저장된 입력 데이터를 출력합니다.
- `_fit_X`에는 `fit()`에 전달한 길이와 무게 데이터가 NumPy 배열 형태로 들어 있습니다.
- 이를 통해 KNN이 훈련 데이터를 내부에 보관한다는 사실을 확인할 수 있습니다.

### 주의

- 이름이 밑줄 `_`로 시작하는 속성은 scikit-learn의 **내부 구현용 속성**입니다.
- 학습 목적으로 확인할 수 있지만 일반 프로젝트에서는 직접 사용하는 것을 권장하지 않습니다.
- 라이브러리 버전이 변경되면 이름이나 동작이 달라질 수 있습니다.

## 14. 모델이 저장한 정답 데이터 확인

```python
print(kn._y)
```

- 앞의 35개 `1`은 도미, 뒤의 14개 `0`은 빙어입니다.
- `_fit_X`와 마찬가지로 밑줄로 시작하므로 내부 확인용 속성입니다.

### KNN의 학습 방식 정리

```text

→ 가장 많은 클래스를 예측값으로 반환
```

## 15. 이웃 수가 49개인 모델 만들기

```python
kn49 = KNeighborsClassifier(n_neighbors=49)
```

### 설명

- 새로운 KNN 분류 모델을 만듭니다.
- 모델 이름은 `kn49`입니다.
- `n_neighbors=49`로 설정해 예측할 때 전체 49개 샘플을 이웃으로 사용합니다.
- 기본값 `5`와 달리 모든 훈련 샘플이 투표에 참여합니다.

```python
kn49.fit(fish_data, fish_target)
```

### 설명

- `kn49` 모델에 같은 생선 데이터와 정답을 전달합니다.
- KNN은 전달받은 데이터를 내부에 저장합니다.

```python
kn49.score(fish_data, fish_target)
```

### 설명

- 전체 데이터를 예측하고 정확도를 계산합니다.

### 실행 결과

```text
0.7142857142857143
```

### 왜 약 71.4%인가?

전체 49마리 중:

```text
도미: 35마리
빙어: 14마리
```

이웃을 49개로 설정하면 어떤 생선을 예측하더라도 전체 샘플을 모두 확인합니다.  
전체에서 도미가 더 많기 때문에 모든 샘플을 도미로 예측하게 됩니다.

```text
올바르게 맞힌 도미 수 = 35
전체 샘플 수 = 49

정확도 = 35 / 49
       ≈ 0.7143
```

따라서 도미는 모두 맞히지만 빙어 14마리는 모두 틀립니다.

### 여기서 알 수 있는 점

- 이웃 수가 많다고 항상 좋은 모델이 되는 것은 아닙니다.
- 너무 많은 이웃을 사용하면 가까운 데이터의 특징보다 전체 클래스 비율의 영향을 받습니다.
- 하이퍼파라미터는 데이터에 맞게 적절하게 선택해야 합니다.

## 16. 정확도가 처음 감소하는 이웃 수 찾기

```python
kn = KNeighborsClassifier()
```

### 설명

- 기본 KNN 분류 모델을 새로 만듭니다.
- 기본 이웃 수는 5입니다.
- 앞에서 사용한 `kn` 변수에 새로운 모델 객체를 다시 저장합니다.

```python
kn.fit(fish_data, fish_target)
```

### 설명

- 새 모델에 입력 데이터와 타깃을 전달해 학습합니다.

```python
for n in range(5, 50):
```

### 설명

- `n`을 5부터 49까지 하나씩 증가시키며 반복합니다.
- `range(5, 50)`에서 마지막 숫자 50은 포함되지 않습니다.
- 실제 반복 값은 `5, 6, 7, ..., 49`입니다.

```python
    kn.n_neighbors = n
```

### 설명

- 현재 반복의 `n` 값을 모델의 이웃 수로 설정합니다.
- 첫 반복에서는 5, 다음 반복에서는 6이 됩니다.
- 이 코드에서는 KNN이 예측 시 이웃 수를 사용하므로 값을 변경해 점수를 비교할 수 있습니다.

```python
    score = kn.score(fish_data, fish_target)
```

### 설명

- 현재 이웃 수로 정확도를 계산합니다.
- 계산한 점수를 `score` 변수에 저장합니다.

```python
    if score < 1:
```

### 설명

- 정확도가 1보다 작은지 확인합니다.
- 정확도 1은 모든 샘플을 맞힌 상태입니다.
- 즉, 처음으로 하나 이상의 샘플을 틀리는 시점을 찾습니다.

```python
        print(n, score)
```

### 설명

- 정확도가 처음 감소한 이웃 수와 그때의 정확도를 출력합니다.

```python
        break
```

### 설명

- 조건을 만족한 순간 반복문을 즉시 종료합니다.
- 이 줄이 없으면 18부터 49까지 정확도가 1보다 작은 모든 결과가 출력됩니다.

### 실행 결과

```text
18 0.9795918367346939
```

### 결과 해석

- 이웃 수가 17개일 때까지는 훈련 데이터를 모두 맞혔습니다.
- 이웃 수가 18개가 되자 정확도가 약 `0.9796`으로 감소했습니다.
- 전체 49개 중 48개를 맞힌 값입니다.

```text
48 / 49 ≈ 0.9796
```

# 💡 이 실습에서 가장 중요한 내용

## 1. 머신러닝 데이터는 입력과 정답으로 구성됩니다

```text
입력 데이터 X = 길이와 무게
정답 데이터 y = 도미 1, 빙어 0
```

모델은 입력과 정답의 관계를 이용해 새로운 입력의 클래스를 예측합니다.

## 2. KNN은 가까운 데이터를 참고합니다

KNN은 별도의 복잡한 수식을 만드는 대신 훈련 데이터를 저장합니다.  
새로운 데이터가 들어오면 기존 데이터와의 거리를 계산해 가장 가까운 이웃을 찾습니다.

## 3. `n_neighbors`는 중요한 하이퍼파라미터입니다

- 너무 작으면 일부 데이터나 잡음의 영향을 크게 받을 수 있습니다.
- 너무 크면 가까운 이웃보다 전체 클래스 비율의 영향을 크게 받습니다.
- 적절한 값은 훈련 데이터가 아닌 검증 데이터 등을 이용해 선택해야 합니다.

## 4. 훈련 점수 100%가 완벽한 모델을 의미하지 않습니다

이 실습의 `1.0`은 훈련한 데이터를 다시 평가한 결과입니다.  
처음 보는 데이터에 대한 성능을 확인하려면 데이터셋을 분리해야 합니다.

# 🔧 더 좋은 반복문 작성 예시

현재 코드도 실습 목적에는 작동하지만, 일반적으로는 각 이웃 수마다 모델을 새로 생성하면 설정과 학습 상태가 명확해집니다.

```python
for n in range(5, 50):
    model = KNeighborsClassifier(n_neighbors=n)
    model.fit(fish_data, fish_target)
    score = model.score(fish_data, fish_target)

    if score < 1:
        print(n, score)
        break
```

각 반복에서:

1. 새로운 모델을 생성합니다.
2. 현재 `n`을 이웃 수로 설정합니다.
3. 데이터를 학습합니다.
4. 정확도를 계산합니다.
5. 정확도가 처음 감소하면 결과를 출력하고 종료합니다.
